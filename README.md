# Исследование и оптимизация каскадной системы детекции и классификации самолетов

Репозиторий содержит выполнение тестового задания по модификации архитектуры YOLOv8, оптимизации нейросетевых моделей и построению двухстадийного пайплайна (Detection + Classification) в среде **ONNX Runtime**.

##  Структура проекта
*   `/models` — .pt и их сконвертированные ONNX модели.
*   `/scripts` — Скрипты обучения и финальный пайплайн.
*   `/configs` — YAML файлы кастомных архитектур (V1, V2).
*   `/results` — Визуализация работы алгоритма.
*   `RESEARCH.md` — Подробный научно-технический отчет.

## Реализованные этапы

### 1. Модификация и обучение детекторов
Было проведено исследование влияния «глубокой ампутации» слоев на производительность. Основные изменения:
*   **Удаление уровня P4:** Исключена ветка детекции средних объектов.
*   **Сокращение слоев:** В версии **Custom V2** удалены промежуточные блоки Backbone (слои 5, 6), что позволило снизить время инференса до **3.8 мс**.
*   **Обучение:** Все модели обучены на 100 эпохах (Dataset: COCO Airplane).

### 2. Каскадная классификация (Stage 2)
Для уточнения типа самолета внедрена вторая стадия обработки:
*   **Датасет:** FGVC-Aircraft (уровень **Family**, 70 классов).
*   **Сравнение:** Обучены **YOLOv8n-cls** (Accuracy: **86.3%**) и **ResNet-50** (Accuracy: **89.4%**).
*   **Выбор:** Для итогового пайплайна выбрана YOLOv8n-cls как наиболее эффективная по соотношению "точность/скорость" (инференс **0.1 ms**).

### 3. Инференс в среде ONNX Runtime
Разработан универсальный пайплайн, работающий без зависимости от PyTorch:
1. Детектор локализует самолет.
2. Программно выполняется **Crop** найденного объекта.
3. Классификатор определяет семейство (напр. *Boeing 737* или *Airbus A320*).
4. Отрисовка финальной рамки с уточненным классом и степенью уверенности.
